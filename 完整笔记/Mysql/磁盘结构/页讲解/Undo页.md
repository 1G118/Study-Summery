用于存储Undo Log（撤销日志）的特殊磁盘页，主要作用是支持事务回滚（ROLLBACK）和MVCC（多版本并发控制）。

>在学习这部分要理解Undo表和数据表区分开来，它不是B+树结构，而是链表结构。

## 存储位置
Undo页的存储位置发生过多次变化。最开始在MySQL5.6之前，Undo页存储在系统表空间里，但由于==这会导致系统表空间会不断的膨胀难以收缩，大小会越变越大。==在MySQL5.6/5.7时支持将Undo表空间独立出去，还需手动配置，而在MySQL8.0之后便默认创建两个独立的Undo表空间，且依旧支持动态的添加更多表空间。

>这里就和磁盘结构里表分类里面串起来了,记不清了回去看一眼。[[InnoDB的磁盘结构]]


## 一. Undo页的两大类

Undo页在InnoDB设计上主要分为
1. INSERT Undo页
2. UPDATE Undo页

### 2.1 File Header

| 字段                          | 大小  | 作用                  |
| --------------------------- | --- | ------------------- |
| FIL_PAGE_SPACE_OR_CHKSUM    | 4B  | 校验和                 |
| FIL_PAGE_OFFSET             | 4B  | 页号（表空间内的偏移量）        |
| FIL_PAGE_PREV/FIL_PAGE_NEXT | 各4B | 双向链表指针（Undo页间串联用）   |
| FIL_PAGE_LSN                | 8B  | 最后一次修改本页的LSN（日志序列号） |
| FIL_PAGE_TYPE               | 2B  | 页类型                 |

### 2.2 Undo Page Header

| 字段                       | 大小  | 作用                     |
| ------------------------ | --- | ---------------------- |
| TRX_UNDO_PAGE_TYPE       | 2B  | Undo类型标记               |
| TRX_UNDO_PAGE_START      | 2B  | 本页第一条Undo Record的起始偏移量 |
| TRX_UNDO_PAGE_FREE       | 2B  | 本页空闲空间起始偏移量，新纪录从这里写入   |
| TRX_UNDO_PAGE_NODE       | 12B | 双向链表节点                 |
| TRX_UNDO_PAGE_LIST_START | 2B  | 本页记录链表的头偏移量            |
### 2.3 Undo Record区
这里存放实际旧数据的区域，从==TRX_UN DO_PAGE_FREE==指向的位置开始写入

#### UndoRecord内部结构

##### 1. 头部信息

| 字段        | 说明                                 |
| --------- | ---------------------------------- |
| next      | 指向下一条（更旧的数据）Undo Record的偏移量，形成单项链表 |
| start     | 记录自身的起始偏移量                         |
| type_cmpl | 编译类型标记，记录操作内类型                     |
| undo_no   | 事务内Undo记录序号                        |
| table_id  | 被修改表所在表空间id                        |
##### 2. 数据体

| INSERT                   | UPDARE/DELETE |
| ------------------------ | ------------- |
| 记录聚集索引的row_id,回滚时按主键精准删除 | 记录被修改列的前镜像    |
|                          | 包含行格式下的字段偏移列表 |
|                          | 包含聚簇索引记录的指针   |
### 2.4 Free Space
Undo Record可分配的空间，可以理解为电脑里面磁盘的剩余空间，当空间不够时，分配新的Undo页

### 2.5 File Trailer
用于检查页是否完整写入磁盘

| 组成部分                | 大小  | 作用                              |
| ------------------- | --- | ------------------------------- |
| Checksum            | 4字节 | 基于页内容计算的数字，用于检查数据是否被篡改或损坏       |
| Log Sequence Number | 4字节 | 记录该页最后一次修改的LSN编码，用于确保日志与数据页的一致性 |
